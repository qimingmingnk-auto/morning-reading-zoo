# 早读动物园 — 产品需求规格（SPEC）

## 1. 解决的问题

**用户**：小学 1-3 年级学生（6-9岁），早读期间声音太小、积极性不高。
**操作者**：班主任/语文老师，在自己笔记本上打开网页，投屏展示给全班。
**痛点**：传统早读缺乏即时反馈，学生没有动机大声朗读。

## 2. 功能描述（核心玩法）

- 老师打开网页，点击"开始监听"，浏览器申请麦克风权限后开始检测环境分贝。
- 当实时分贝值超过老师设置的**分贝阈值**时，屏幕上按照设定的**生成速度**持续出现随机动物 Emoji。
- 当声音低于阈值时，**停止新增动物**，已有动物**缓慢移出屏幕**（速度很慢，保留成就感）。
- 页面显示：当前分贝值、分贝阈值、已持续大声朗读的计时器、动物展示区。
- **设置面板**（老师课前配置）：
  - 分贝阈值：滑动条调节（默认 50dB）
  - 动物生成速度：滑动条，最快1秒/只 ～ 最慢60秒/只
  - 动物种类：勾选启用/禁用某类 Emoji 动物（至少选一种）
  - 重置默认 / 保存设置 按钮
- **重置**：刷新浏览器页面即清空所有动物，无需额外重置按钮。
- **分贝与速度映射**：不自动映射，由老师手动设定，产品不做动态调节。

## 3. 技术约束

- **平台**：纯 Web 网页，无需安装，Chrome 浏览器优先（老师笔记本）。
- **麦克风**：使用浏览器 Web Audio API，首次打开会弹出权限申请弹窗（正常行为，提前告知老师）。
- **动物图片**：使用 Unicode Emoji，无需额外图片资源。
- **界面语言**：全中文，UI 风格童趣、色彩鲜明。
- **运行环境**：老师笔记本 + 投屏，无需适配移动端。

## 4. 明确不做的事

| 不做 | 原因 |
|------|------|
| 用户登录/账号 | 无需身份识别 |
| App（iOS/Android） | 仅做 Web |
| 数据持久化 | 无需保存历史记录 |
| AI 功能 | 超出范围 |
| 计分/排行榜 | 避免功能蔓延 |
| 声音录制/回放 | 隐私风险且不必要 |
| 个别学生声音识别 | 技术复杂且不在范围 |

## 5. 成功标准

1. **老师上手**：打开浏览器，30 秒内完成配置并启动监听，无需说明书。
2. **学生反应**：学生看到动物出现后，可观察到班级整体朗读音量明显提升。
